⚙️ Quản lý dữ liệu Cycle Results

Chỉ bạn (admin) mới có quyền sửa/xóa dữ liệu nhập vào. Tìm kiếm theo tên script, model, hoặc người nhập.

Mẹo: Tìm kiếm cực nhanh - nhập tên script, model, hoặc tên người nhập rồi bấm "Tìm kiếm"

# Cycle Test Suite Test Case Model State Result Author (Owner) Team Người nhập Ngày nhập Hành động

✍️ Quản lý dữ liệu Script viết mới

Xem, sửa, xoá từng dòng, hoặc nhập hàng loạt (dán từ Excel) cho bảng "Script viết mới".

Mẹo: Tìm theo TC ID, Item, Member, hoặc Team rồi bấm "Tìm kiếm".

# Item TC ID Member Team Week Completed date Status Model đã viết SDF ID Remark Hành động

📋 Nhập hàng loạt (dán từ Excel)

Dán nhiều dòng, các cột phân cách bằng Tab (copy thẳng từ Excel), theo đúng thứ tự:
TC ID  Member  Status(DONE/SKIP)  Completed date(YYYY-MM-DD)  Model đã viết (phân cách bằng dấu phẩy hoặc /)  SDF ID  Remark
Dòng nào lỗi (TC ID sai định dạng, trùng, SKIP thiếu Remark...) sẽ bị bỏ qua và báo rõ lý do — các dòng hợp lệ khác vẫn được nhập.

🛠️ Quản lý dữ liệu Theo dõi Fix

Thêm / sửa / xóa trực tiếp các lần fix đã ghi nhận. Tìm kiếm theo owner, test suite, test case, model, hoặc root cause.

Mẹo: Bấm "➕ Thêm fix mới" để tạo 1 dòng fix tay (VD: bù dữ liệu bị thiếu), hoặc dùng khối "Nhập hàng loạt" bên dưới để nhập nhiều dòng cùng lúc. Sửa/xóa hoạt động như bảng Cycle Results.

# Ngày fix Owner Test Suite Test Case Model đã fix Fix sau cycle Root cause Ghi chú Tạo lúc Hành động

📋 Nhập hàng loạt (dán từ Excel)

Dán nhiều dòng, các cột phân cách bằng Tab, theo đúng thứ tự:
Fix date(YYYY-MM-DD)  Owner  Test suite  Test case  Model fixed  Fixed after cycle  Root cause  Note
Trùng (owner + test suite + test case + model fixed + fixed after cycle) sẽ cập nhật dòng cũ thay vì tạo dòng mới, giống ghi nhận fix bình thường.

👤 Quản lý dữ liệu Assignments (đang phụ trách)

Xem, sửa, xoá từng dòng, hoặc nhập hàng loạt danh sách người đang phụ trách từng script.

Mẹo: Tìm theo Test suite, Test case, hoặc Owner rồi bấm "Tìm kiếm". Sửa/xoá ở đây chỉ đổi người đang phụ trách hiện tại, không ảnh hưởng lịch sử Fix Log.

Test suite Test case Owner Assigned date Hành động

📋 Nhập hàng loạt (dán từ Excel)

Dán nhiều dòng, các cột phân cách bằng Tab, theo đúng thứ tự:
Test suite  Test case  Owner  Assigned date(YYYY-MM-DD, để trống = hôm nay)
Trùng (test suite + test case) sẽ ghi đè người phụ trách hiện tại (upsert).

👥 Quản lý tài khoản & phân quyền

Đổi loại tài khoản (admin / moderator / user), bật/tắt từng quyền chức năng, reset mật khẩu (về smartlab1@), vô hiệu hoá và xoá tài khoản. Tài khoản chỉ xoá được khi đã ngừng hoạt động.

Lưu ý: Tài khoản đăng ký mới mặc định là user. Đổi loại tài khoản sẽ đặt lại bộ quyền về mặc định của loại đó; sau đó có thể tick/bỏ từng quyền. Người dùng đang đăng nhập sẽ thấy thay đổi trong vòng ~15 giây.

Tài khoản Loại Quyền chức năng Trạng thái Hành động

🧑‍🤝‍🧑 Quản lý Owner & Team

Thêm, sửa (đổi tên / đổi team), ngừng hoạt động hoặc xoá hẳn owner. Đổi tên/ngừng/xoá sẽ tự đồng bộ với tài khoản đăng nhập tương ứng.

Lưu ý: "Xoá hẳn" chỉ được phép khi owner không còn dữ liệu tham chiếu (kết quả / fix / assignment). Nếu còn, hãy dùng "Ngừng hoạt động" để giữ lịch sử.
