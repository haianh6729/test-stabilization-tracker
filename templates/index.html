    📥 Nhập kết quả chạy (dán từ Excel / test farm export)

    Dán dữ liệu theo đúng thứ tự cột — hỗ trợ cả 2 định dạng: 6 cột (Test ID, Model, Test suite, Test Case, State, Description) hoặc 8 cột (Request ID, Model, Test Suite, Test Case, State, Description, Author, Team) — copy thẳng từ Excel/SDF export là được (tab phân cách). Test Suite/Test Case sẽ được hệ thống tự nhận diện và chuẩn hoá tên (VD: Internet/test suite/Internet_ui90_part4.ts hoặc Browser_000118.py đều tự suy ra đúng Test Suite là Internet, Test Case là Browser_000118). State RUNNING sẽ tự động bị bỏ qua (chưa chạy xong).

    🗓️ Cycle tự động theo ngày: hệ thống lấy ngày tạo từ Test ID (VD 260706-xxx = 06/07/2026). Các Test ID cùng ngày = cùng 1 cycle; ngày sau = cycle kế tiếp. Bạn không cần nhập cycle thủ công. (Nếu Test ID không mã hoá ngày, hệ thống dùng ngày chạy nhập tay bên dưới.)

    🛠️ Ghi nhận đã fix xong 1 script

    Mỗi thành viên tự điền ngay khi fix xong. Chọn trực tiếp script đang fail từ danh sách bên dưới — hệ thống tự điền Test suite / Test Case / Model đang lỗi / Fixed_after_cycle. Sau khi ghi nhận, vào tab 🔬 Theo dõi Fix để xem lần chạy sau đã hết lỗi chưa.

    🔄 So sánh Script qua từng Cycle

    Mỗi dòng là 1 script (test suite / test case), gộp mọi model đã chạy. Mỗi cột Cycle hiển thị Pass Rate = (Pass+Check+Manual Check) / (Tổng − NA), kèm số Fail/Tổng lượt chạy. Mũi tên so với lần GẦN NHẤT TRƯỚC ĐÓ mà script này có chạy: ▲ tốt lên · ▼ xấu đi · = không đổi. Cột "Xu hướng" tổng kết 2 lần chạy gần nhất để biết script nào vừa cải thiện hay vừa mới giảm sút.

    🔬 Theo dõi Fix — test case đã báo fix thì hết lỗi chưa?

    Mỗi dòng là 1 lần fix đã ghi nhận. Hệ thống đối chiếu với các lần chạy SAU cycle fix để tự đánh giá: ✅ Đã hết lỗi (chạy lại Pass và giữ nguyên) · ⚠️ Hết rồi fail lại (Pass ngay sau fix nhưng cycle sau tái lỗi) · ❌ Chưa hết lỗi (ngay sau fix vẫn Fail) · ⏳ Chờ dữ liệu (chưa có lần chạy sau fix để đối chiếu).

    🚜 Lấy kết quả từ Test Farm theo Test ID
    
    
    Nhập danh sách Test ID (mỗi dòng 1 ID, hoặc cách nhau bằng dấu phẩy) → hệ thống gọi API farm lấy kết quả và nhập thẳng vào dữ liệu (đi qua đúng bộ chuẩn hoá/chống trùng như dán tay). Cấu hình URL/token API farm ở tab ⚙️ Cài đặt (mục Tích hợp & API) — khi chưa cấu hình sẽ báo rõ. Ngoài ra script phía farm có thể tự push kết quả qua POST /api/results/import với header X-Import-Token.

    🎯 Tiêu chí & KPI
    
    
    Exit criteria: script chỉ tính "Done" khi mọi model pass đủ N cycle liên tiếp gần nhất (N=1 = hành vi cũ, chỉ xét cycle mới nhất). Flaky: script đổi pass↔fail ≥ số lần cấu hình trong cửa sổ N cycle gần nhất. Loại script mới: >0 sẽ thêm KPI "Pass rate (bỏ script mới)" không tính script trong N cycle đầu của nó.
